# Anker Atlas Relay — Product System Architecture V1

## 1. 产品重新定义

**Atlas Relay 不是万能旅行插座，也不是电压转换器。**

它是一套面向高频跨国、多设备旅行者的 USB-C 供电系统：

> 通过地区短电源线把重型 GaN 主机从墙面移到桌面，并记录 AC 中断、USB-C 重连和异常低功率事件，让用户知道整次夜间供电是否稳定完成。

## 2. MVP 目标规格（工作假设）

| 项目 | V1 目标 |
|---|---|
| 输入 | 100–240V AC, 50/60Hz |
| 产品类别 | Class II 双重绝缘 USB-C 电源适配器 |
| AC 透传 | 无 |
| 电压转换 | 无 |
| 地区组件 | US / EU / UK / AU 四种短电源线，按行程选带 |
| 输入线长度 | 30cm、60cm、90cm 三档实验，最终选一档或两档 |
| 主机输出 | 3 × USB-C |
| 总输出目标 | 100W |
| 单口目标 | C1 最高 100W；具体多口分配待热设计与用户任务验证 |
| 本机反馈 | 状态灯/小屏 + 可关闭蜂鸣 |
| 无线连接 | 可选 BLE，不强制注册账号 |
| 电池 | 仅保留小型超级电容做断电“last-gasp”，不含大容量充电宝 |
| 目标用户 | 笔记本 + 手机 + 至少一台相机/平板/配件的跨国旅行者 |

## 4. 硬件模块

### 4.1 Country Cord Module

推荐形态不是厚重墙插模块，而是：

- 当地标准插头；
- 低轮廓外壳；
- 30–90cm 柔性双芯线；
- 主机端锁定式低压差 AC 连接器，或经过认证的 IEC appliance coupler 方案；
- 每条地区线独立满足当地插头、保险和绝缘要求。

设计目标：

- 墙端只承受插头和少量线材重量；
- 不包含 USB 电子模块；
- 不包含万能 AC 母座；
- 减少滑动插脚和复杂活动件；
- 允许只携带本次行程需要的地区线。

### 4.2 Input Protection

概念级模块包括：

- 输入保险/保护元件；
- 浪涌抑制；
- EMI 滤波；
- 浪涌电流限制；
- 过温保护；
- 高压母线电压检测。

具体元件和认证方案由安克电源团队决定，比赛阶段不锁定 BOM。

### 4.3 Isolated GaN AC-DC Stage

目标：

- 100–240V 宽电压输入；
- 约 100W 总输出；
- 隔离式 AC-DC；
- 为后级多口 DC-DC 提供中间母线；
- 保留足够热余量，不追求极限体积。

USB-IF 的 USB PD 3.1 可支持高于 100W 的 EPR，但当前 MVP 选择约 100W，是为了在旅行重量、热设计和目标设备之间取得平衡。

### 4.4 Multi-Port USB-C Stage

每个端口需要：

- USB-C / PD 控制器；
- 独立或半独立 DC-DC 路径；
- 电流、电压和功率测量；
- 端口插拔和协商事件；
- OCP / OVP / SCP 等保护。

建议第一版评估：

- 单口：100W；
- 双口：65W + 35W；
- 三口：45W + 30W + 25W；

这只是产品定义目标，不是已经完成的电源设计。

### 4.5 Charge Assurance Controller

系统 MCU 汇总：

- AC 输入中断/恢复；
- 高压母线 brownout；
- 每个 USB-C 端口的连接、断开和 PD 合约变化；
- 实时输出 V / I / W；
- 每端口累计 Wh；
- 温度与保护事件；
- 多口重新分配是否造成非预期中断。

### 4.6 Last-Gasp Power

当 AC 突然断开时，普通充电器会立即失去通信能力。V1 可使用小型超级电容，为 MCU 提供数秒能量，以完成：

1. 写入非易失事件记录；
2. 点亮/蜂鸣短暂告警；
3. 若手机就在附近，尝试发送一次 BLE 断电通知。

不使用大容量锂电池，避免把产品重新变成充电宝，并降低运输、老化和认证复杂度。

## 5. “充电完整性确认”的真实定义

### 能直接确认

- 主机是否失去 AC 输入；
- AC 是否发生短时中断和恢复；
- USB-C 端口是否物理断开；
- PD 合约是否反复重新建立；
- 协商的电压/电流档位；
- 实际输出电压、电流、功率和累计能量；
- 端口是否长时间显著低于合理输出；
- 主机是否过热、限功率或触发保护。

### 只能推断，不能确诊

- 是线材、插座、设备还是协议导致低功率；
- 设备是否因系统策略主动降速；
- 某台设备是否已经充满；
- 插座内部夹持力是否不足；
- 用户明早是否一定“就绪”。

### 默认不能知道

- 所有手机、相机、耳机和电脑的精确电池百分比；
- 设备内部电池健康状态；
- 非标准设备的充电策略；
- 未连接到产品 App 的行程安排。

因此 V1 的营销语言应是：

> “确认供电会话是否持续、稳定并按预期输出。”

而不是：

> “保证所有设备明早满电。”

## 6. 软件与交互

### 本机优先

- 绿色：本次会话稳定；
- 黄色：发生过中断、重连或明显低功率事件；
- 红色：当前无输入、过温或保护；
- 端口页：显示当前功率和累计 Wh；
- 事件页：显示最近一次异常类型。

### 可选 App

- 选择旅行国家并提示携带地区线；
- 保存设备类型和大致功率需求；
- 查看异常时间线；
- 设置夜间蜂鸣开关；
- 固件升级。

App 不应成为基本充电功能的前提。

## 7. 安全与合规边界

概念阶段建议以以下方向为约束，但不得声称已经认证：

- USB-IF USB Type-C / USB Power Delivery；
- IEC/UL 62368-1 类电源产品安全要求；
- 各地区插头和电源线标准；
- EMC / FCC / CE 等市场准入；
- RoHS / REACH 等材料要求；
- Class II 双重绝缘；
- 足够爬电距离、电气间隙和阻燃材料；
- 每条地区电源线单独验证额定电流、机械强度和弯折寿命。

## 8. 为什么不做 AC 万能透传

取消 AC 万能母座可以：

- 降低墙端结构和活动件复杂度；
- 避免用户接入吹风机、电热水壶等高功率、非宽电压电器；
- 降低接地与地区安全歧义；
- 减少体积；
- 让产品聚焦 USB-C 多设备供电。

## 9. 工程 Kill Criteria

以下任一结果成立，Atlas Relay 应降级或否决：

1. 实体实验显示稳定型短机身壁插与短线方案差异很小；
2. 地区线和主机总重量明显高于同功率竞品，且用户不愿接受；
3. 断电告警需要大电池或导致显著成本、误报；
4. 30–90cm 线材被多数目标用户认为比掉落风险更麻烦；
5. 多口独立不中断架构在 100W 体积内不可实现或温升过高；
6. 大规模评论编码显示机械失效仅是极少数劣质插座问题。
