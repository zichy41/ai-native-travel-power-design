# Opportunity Map / 机会地图

## 1. 两条关键轴

### 轴 A：墙端机械架构

```text
完整高功率主机挂墙  ←────────────→  墙端仅保留轻量插头
```

### 轴 B：旅行地区适配策略

```text
万能机械结构  ←────────────→  行程选择的轻量地区模块
```

市场样本主要集中在：

- 万能一体机 + 完整机身挂墙；
- 可换插脚 GaN + 完整主机挂墙；
- 固定地区短机身 + 壁插稳定优化；
- 桌面主机 + 长电源线。

Atlas Relay 的目标位置是：

> 行程选择的轻量地区模块 + 短线卸载 + 桌面高功率主机。

## 2. 第三条差异轴：可靠性反馈

大多数样本提供：

- 通电 LED；
- 温度保护；
- 保险丝；
- 功率分配。

当前产品假设希望增加：

- AC 输入是否反复中断；
- USB-C 端口是否反复重新协商；
- 输出是否长时间显著低于合理区间；
- 是否在计划时间前完成“可验证的供电任务”。

注意：

> 主机不一定能读取每台设备的真实电池百分比。第一版应把“就绪”定义为稳定供电时长、端口能量输出和异常事件，而不是承诺读取所有设备 SOC。

## 3. 目标用户进一步收窄

### Primary Segment

高频跨国、多设备旅行者：

- 携带笔记本；
- 至少两台 USB-C 移动设备；
- 经常入住老酒店、民宿、青旅或山屋；
- 对第二天设备可用性敏感；
- 愿意为可靠性而不是极限功率付费。

### Excluded Segment

- 只带手机的极简旅行者；
- 很少出国、可接受临时购买转换头的人；
- 需要吹风机、电热水壶等高功率电压转换的人；
- 追求 140W+ 单口且愿意承担更大体积的人。

## 4. 产品价值主张

传统竞品：

> 一个设备覆盖更多国家、更多端口、更高功率。

Atlas Relay：

> 在陌生插座里，减少墙端机械失效，并让用户知道整夜供电是否真正持续完成。

## 5. 当前核心功能优先级

| 层级 | 功能 | 说明 |
|---|---|---|
| Must | 轻量地区插头 | 控制墙端质量和悬臂 |
| Must | 25–30cm 短柔性输入线 | 让主机落桌，适配遮挡和凹陷插座 |
| Must | 100W 左右、多 USB-C 主机 | 覆盖笔记本 + 手机 + 相机等 |
| Must | 输入中断事件记录 | 捕捉半夜插头掉落或接触不良 |
| Should | 异常低功率与反复协商提示 | 提醒线材、接口或接触问题 |
| Should | 本机状态显示 | 不依赖 App |
| Could | 行程模块推荐 | 减少带错插头 |
| Could | 手机通知 | 可选，不强制账号 |
| Later | 可拆卸电池 | 证据不足，V2 再验证 |
